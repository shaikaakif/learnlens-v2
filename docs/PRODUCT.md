# Product Specification

## Vision
Turn every assessment from an endpoint into a diagnostic event.

Every analysis should answer:
1. What happened?
2. Why did it happen?
3. Is the pattern recurring?
4. What should the learner do next?
5. Did the learner improve later?

## Core Modules
### Authentication
Roles: Student, Teacher, Parent, School Admin.

### Assessment Management
Teacher creates an assessment and uploads question paper plus optional rubric/marking scheme.

### Answer Sheet Capture
Student takes photos or uploads images/PDF of the corrected answer sheet.

### AI Analysis
The backend combines the answer sheet with stored assessment context.

### Learning MRI
Shows strengths, improvement opportunities, mark-loss patterns, evidence, confidence, and next actions.

### What Next?
Generates a portable personalized tutoring context for use with an external AI tutor.

### Learning Profile
Stores structured historical diagnostic findings.

### Teacher Dashboard
Aggregates class patterns and common misconceptions.

### Parent View
Explains performance in simple, actionable language.

## Diagnostic Categories
Concept misunderstanding; partial understanding; recall gap; application difficulty; question misinterpretation; command-word misunderstanding; incomplete answer; irrelevant content; weak structure; missing key points; spelling; grammar; punctuation; capitalization; handwriting ambiguity; calculation; formula selection; units; sign errors; diagram/labeling; skipped question; possible time-management issue; recurring error.

## Product Success
A student moves from:
“I got 31/40.”
to:
“I understand why I got 31/40, what pattern caused the loss, and what I should change before the next exam.”
